Przetwarzanie zbioru danych

Odczyt danych

Odczyt danych przeprowdzony jest bezorednio ze strony, aby zapeweni sta warto pocztkow dla kadego uruchomienia. Dodatkowo uzupeniane s brakujce dane identyfikatory pacjentw PATIENT_ID. Wartoci brakujce wynikaj ze sposobu przetwarzania scalonych komrek pliku rdowego.

# https://stackoverflow.com/a/41368947
url <- "http://www.cs.put.poznan.pl/dbrzezinski/teaching/zed/wuhan_blood_sample_data_Jan_Feb_2020.xlsx"
GET(url, write_disk(tf <- tempfile(fileext = ".xlsx")))
df <- read_excel(tf) %>% fill(PATIENT_ID)

Atrybuty zbioru danych

Surowy zbir danych zawiera 81 atrybutw i 6120 wierszy.

Atrybuty oglne

Pierwszych siedem kolumn to atrybuty oglne, reprezentuj nastpujco:

  • PATEINT_ID - sztuczny identyfikator pacjenta
  • RE_DATE - najpewniej jest to data wprowadzenia wyniku bada prbki krwi do systemu
  • age - wiek pacjenta
  • gender - pe pacjenta, przyjmuje warto 1 albo 2, ale z poisu zbioru danych nie wynika wprost, ktre oznaczenie jest przypisane do ktrej pci
  • Admission time - czas przyjcia pacjenta do szpitala
  • Discharge time - czas zakoczenia leczenia (w skutek wypisu pacjenta lub jego mierci)
  • outcome - rezultat, przyjmuje warto 0 jeeli pacjent zosta wyleczony, lub warto 1 jeeli zmar.

Nazwy wyej wymienionych atrybutw zostan przeformatowane, bd pisane wielkimi literami, a ewentualne spacje bd zamieniane na znak podkrelinka _, w ten sposb atrbuty oglne bd atwo odrnialne od pozostaych atrybutw.

df <- df %>% rename_with(~ toupper(gsub(" ", "_", .x, fixed = TRUE)), 1:7)

Pola GENDER i OUTCOME zostan dodatkowo zmienione na typ factor - z typu numerycznego zamienione zostan na zmienne nominalne. Do tego trzeba przeprowadzi identyfikacj typu pci.

Wartoci pci nie s jawnie opisane, w artykule rdowym zostaa natomiast podana proporcja pacjentw danej pci, co powinno umoliwi okreslenie waciwego przypisania.
